Basic Steps There is only tree atomic moves:

s (step forward)

Slides the bug one field forward

r (turn right)

Turns the bug right

l (turn left)

turns the bug left

example

So knowing this, it is not too hard to discover that code 

ssrsls

Functions Call the same sequence of steps more than once

Creating code only from atomic moves could be quite hard to understand and not very useful if you want to beat the code counter. Most case maps contained in games are some sort of symmetric structures which means that your code in most cases could be re-used. Instead of using Ctrl-C and Ctrl-V operations we will use functions. 

f:sssr
lf

Analyzing this code from second line. First element is a well known atomic step l (turn left) which means that the first action taken by the bug will be turning left. Next step is f which is not a valid step, but in first line holds a definition of this not atomic step (function). So typing f runs the steps specified in the f function, sssr.

Functions have to be named with the use of single lowercase letters, but not one of the reserved step letters: s, r and l.

You are able to call defined functions as many times and define as many functions as you need (well not quite because Latin alphabet has 24 characters and 3 of them are used for atomic moves, but it is still a lot!). 

f:sssr
x:lsss
rxff

This example will result in 

rlssssssrsssr

Recursion Call the same sequence of steps more than once.
